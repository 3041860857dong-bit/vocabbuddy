## 📌 TL;DR（执行摘要）

VocabBuddy 是一款面向大学生与职场人的英语单词学习 App，以**碎片化时间**为使用场景，以 **SRS 间隔复习算法**为技术核心，帮助用户用最少的重复次数将单词从"短期记忆"推进到"长期掌握"。MVP 围绕 **新词闪卡学习 → SRS 自适应复习 → 生词本强化 → 学习统计** 四大闭环构建，默认每日 15 个新词 + 系统自动调度复习量，单次会话可在 3 分钟内完成有意义的交互。产品主打三大差异化：**透明可调的自适应 SRS 引擎、柔性断链恢复机制、掌握度分层的进度反馈**。MVP 首要目标为 D1 留存 ≥25%、D7 留存 ≥12%、北极星指标（日均有效学习单词数）≥60 词/天。

## 1. 产品概述

### 1.1 目标用户

基于瑞思的用户研究综合报告，VocabBuddy 的目标用户可细分为三个画像：

#### 画像 A：备考冲刺型大学生（占比 45-50%）

| 维度 | 描述 |
|------|------|
| **人群特征** | 18-24 岁，本科/研究生在读，备考 CET-4/6、考研英语、雅思、托福等 |
| **使用场景** | 早晚自习、课间、图书馆，每日 30-60 分钟连续学习 |
| **行为特征** | 高频深度用户，考前 1-3 个月达到使用峰值，考后回落 |
| **核心诉求** | 词库须覆盖考试大纲；学习计划可量化（"我今天背了多少、还剩多少"）；效率优先 |
| **典型用户故事** | "As a 考研学生，我想在 3 个月内掌握 5500 考研词汇，这样我才能在英语阅读中不被词汇量拖后腿。" |

#### 画像 B：职场提升型白领（占比 30-35%）

| 维度 | 描述 |
|------|------|
| **人群特征** | 25-35 岁，一线城市上班族，有职场英语需求（邮件、会议、商务沟通）或自我提升意愿 |
| **使用场景** | 地铁通勤、午休、睡前，每日 10-20 分钟碎片时间 |
| **行为特征** | 中频用户，使用频次不稳定，容易被工作打断 |
| **核心诉求** | 快速进入退出（"3 分钟能学完一轮"）；不要有负担感；能坚持下来 |
| **典型用户故事** | "As a 上班族，我想在地铁通勤的 15 分钟里背几个单词，这样我不会觉得自己在浪费碎片时间。" |

#### 画像 C：兴趣驱动型终身学习者（占比 15-20%）

| 维度 | 描述 |
|------|------|
| **人群特征** | 22-40 岁，有阅读英文原著/看无字幕影视/旅行交流等真实使用场景 |
| **使用场景** | 不规律，低频长尾，可能集中在周末或特定兴趣触发时 |
| **行为特征** | 低频但忠诚度高，重视学习质量而非数量 |
| **核心诉求** | 例句质量高、语境化学习；能手动添加自己遇到的生词 |
| **典型用户故事** | "As a 英语原著爱好者，我想在遇到生词时能快速加入学习列表，之后通过 SRS 记住它，这样下次阅读时就不会卡壳了。" |

### 1.2 核心痛点

基于瑞思的用户研究报告，按严重度从高到低排序：

| 序号 | 痛点 | 严重度 | 数据支撑 | VocabBuddy 对应方案 |
|------|------|--------|---------|-------------------|
| 1 | **背了就忘** | ★★★★★ | 艾宾浩斯曲线：学习后 1 天仅剩 33%；App Store 差评 30-40% 与遗忘相关 | SRS 自适应间隔复习引擎（核心差异化） |
| 2 | **难以坚持** | ★★★★★ | 动机衰减 + 断链重启困难，积压复习形成"债务" | 柔性断链恢复机制（积压分摊 + 补卡 + 回归首日降负荷） |
| 3 | **机械记忆缺语境** | ★★★★☆ | 认识单词但不会用 | 闪卡包含例句语境；拼写练习强化产出能力 |
| 4 | **复习安排不科学** | ★★★★☆ | 复习量忽多忽少，一刀切 | SRS 可调节 + 每日复习量有可预期范围 + 防雪崩机制 |
| 5 | **学习反馈不清晰** | ★★★☆☆ | 不知道自己进步了没有 | 掌握度分层统计（新学→熟悉→掌握→长期记忆） |
| 6 | **词库不匹配** | ★★★☆☆ | 想学的词不在预置词库中 | 生词本支持手动添加 + 自动融入 SRS 队列 |

### 1.3 产品定位

> **VocabBuddy — 用碎片化时间，靠 SRS 间隔复习，把英语单词高效背进长期记忆。**

三个关键词的内涵：
- **碎片化时间**：3 分钟可完成一轮有意义的学习交互，随时进出，进度自动保存
- **SRS 间隔复习**：基于记忆科学的自适应间隔重复算法，让用户在最少的重复次数下实现长期记忆
- **高效背单词**：以"已掌握词数"而非"已学词数"为北极星指标，追求真实学习效果

## 2. MVP 功能范围（功能清单）

### 2.1 账户 / 初始化

#### 功能描述

用户首次打开 App 后的引导流程，包括：
1. **游客模式**：用户可不注册直接进入体验，所有学习数据本地存储；注册后可绑定云端备份（注：MVP 仅本地存储，云备份为 P2）
2. **选择词库**：从预置词库列表中选择目标词库（如 CET-4、CET-6、考研核心、雅思核心、托福核心、商务英语核心等），支持切换
3. **设定每日学习量**：设定每日新词数量（默认 15 个，可在 10-25 之间调节）和每日复习上限（默认 100 个）
4. **设定提醒时间**：选择每日推送提醒的时间（默认 20:00，可自定义）
5. **发音偏好选择**：选择美式或英式发音（默认美式）

#### 验收标准

- [ ] 首次启动 App 进入引导流程，3 步内完成初始化（选词库 → 设参数 → 开始学习）
- [ ] 游客模式可直接进入学习，无强制注册弹窗
- [ ] 词库列表至少包含 6 个预置词库（CET-4、CET-6、考研、雅思、托福、商务英语）
- [ ] 每日新词量可调节，范围 10-25，步进 5，默认值 15
- [ ] 每日复习上限可调节，范围 50-200，步进 10，默认值 100
- [ ] 初始化完成后进入学习首页，显示今日学习计划摘要
- [ ] 所有设置项可在「设置」页面后续修改

#### 优先级：P0

---

### 2.2 新词学习

#### 功能描述

用户每日学习当日新词的闪卡交互流程：
1. **闪卡正面**：显示单词拼写 + 音标 + 发音按钮（点击播放 TTS）
2. **闪卡翻转**：用户点击卡片或滑动，翻转显示释义 + 词性 + 例句（含例句翻译 + 例句发音）
3. **熟悉度标记**：用户翻转后对单词进行自评标记：
   - **"认识"**：已掌握，跳过后续闪卡阶段，直接进入 SRS 队列（间隔 4 天后复习）
   - **"不熟"**：需要复习，进入 SRS 队列（间隔 1 天后复习），同时自动加入生词本
   - **"不会"**：完全陌生，进入 SRS 队列（间隔 1 天后复习），自动加入生词本，并标记为重点
4. **进度自动保存**：每个单词学习后进度立即保存，随时退出，下次从断点续接
5. **学习完成反馈**：当日新词全部学完后，显示本轮学习摘要（新学词数、加入生词本词数）

#### 设计考量（融入洞察 1 & 2）

- **主动提取优先**：即使是新词学习，闪卡正面先呈现单词刺激，鼓励用户先尝试回忆释义，再翻转揭晓——而非直接展示全部信息
- **可中断续接**：单个单词学习周期 ≤30 秒，2 分钟可完成 4-6 个新词学习，退出后进度不丢失
- **例句语境**：每个单词至少包含 1 条例句，帮助用户在语境中理解词义

#### 验收标准

- [ ] 闪卡正面显示：单词拼写、音标、发音按钮
- [ ] 闪卡翻转显示：中文释义、词性、至少 1 条英文例句（含中文翻译）
- [ ] 发音按钮点击播放 TTS，支持美式/英式切换
- [ ] 翻转后显示三个标记按钮：「认识」「不熟」「不会」，用户必须选一个才能进入下一个
- [ ] 标记「认识」→ 该词 SRS 首次间隔设为 4 天
- [ ] 标记「不熟」或「不会」→ 该词 SRS 首次间隔设为 1 天，同时加入生词本
- [ ] 标记「不会」的词在生词本中标记为重点（priority = high）
- [ ] 每学完 1 个词，进度自动保存到本地
- [ ] 中途退出再进入，从上次未完成的词继续
- [ ] 当日新词全部学完，显示学习摘要弹窗

#### 优先级：P0

### 2.3 SRS 复习

#### 功能描述

用户每日复习到期单词的核心交互流程，这是 VocabBuddy 的功能核心和差异化重点：

1. **按日拉取到期词**：系统每天自动计算当日到期的复习词列表，按"最可能遗忘的词优先"排序
2. **翻转自测（先考后看）**：
   - 闪卡正面呈现单词（刺激）→ 用户先尝试回忆释义（提取过程）→ 点击翻转揭晓答案
   - 翻转后显示：释义、词性、例句、上次评分、当前间隔、下次复习时间预览
3. **四档评分**：用户根据回忆质量自评：
   - **Again（重来）**：完全忘记/想不起来 → 间隔重置为 1 天（次日再现）
   - **Hard（困难）**：勉强记得，回忆吃力 → 间隔 × 1.2
   - **Good（良好）**：正常记得，有短暂犹豫 → 间隔 × 2.5（标准增长倍数）
   - **Easy（轻松）**：秒答，毫无犹豫 → 间隔 × 3.0~4.0
4. **即时反馈**：评分后显示该词下次复习日期，增强用户的"掌控感"
5. **复习完成**：当日到期词全部复习完，显示复习摘要（复习词数、各档比例、准确率）

#### 设计考量（融入洞察 1 & 4 & 5）

- **先考后看是必须环节**：不提供"直接看答案"的捷径，翻转前有短暂等待鼓励回忆
- **掌握度可视化**：每个词显示当前所属阶段（新学/熟悉/掌握/长期记忆），让用户看到进步
- **生词本融合**：生词本中的词也在 SRS 队列中，复习流程统一管理，不单独跳转

#### 防雪崩机制（融入竞析差异化机会 2）

- 当日到期复习词超过 80 个时，系统自动触发"复习负载管理"：
  - 按遗忘概率排序，优先复习最可能遗忘的词
  - 超出部分自动分摊到未来 2-3 天
  - 系统建议暂停当日新词学习
- 用户可在设置中开启"复习优先模式"：当日复习量 > 50 时自动暂停新词

#### 验收标准

- [ ] 每日首次打开 App，自动拉取当日到期复习词列表
- [ ] 复习词列表按遗忘概率从高到低排序
- [ ] 闪卡正面仅显示单词，不显示释义（先考后看）
- [ ] 翻转后显示：释义、词性、例句、上次评分、当前间隔天数、评分后下次复习日期
- [ ] 四档评分按钮：Again / Hard / Good / Easy，每个按钮下方预览下次复习间隔
- [ ] 点击评分后，该词间隔按规则更新，移入下一个到期日队列
- [ ] 评分 Again → 间隔重置为 1 天
- [ ] 评分 Hard → 间隔 = 当前间隔 × 1.2（向上取整）
- [ ] 评分 Good → 间隔 = 当前间隔 × 2.5（向上取整）
- [ ] 评分 Easy → 间隔 = 当前间隔 × 3.0~4.0（向上取整，具体系数后续定）
- [ ] 当日到期词 > 80 个时，触发防雪崩：优先复习高风险词，超出部分分摊
- [ ] 防雪崩触发时，提示用户"今日复习量较大，已为您分摊部分到后续几天"
- [ ] 复习完成显示摘要：复习词数、Again/Hard/Good/Easy 各档占比

#### 优先级：P0

### 2.4 生词本 / 收藏

#### 功能描述

生词本是 VocabBuddy 的第二大核心模块，区别于竞品的独立"生词列表"，VocabBuddy 的生词本**与 SRS 深度融合**：

1. **多来源收词**：
   - 新词学习中标记「不熟」或「不会」→ 自动加入生词本
   - SRS 复习中评 Again → 自动加入生词本（如尚未在生词本中）
   - 拼写练习答错 → 自动加入生词本
   - 用户手动添加任意单词 → 加入生词本
2. **生词与 SRS 融合**：
   - 加入生词本时，自动设置该词 SRS 间隔为 1 天（次日即复习）
   - 生词本中的词在 SRS 队列中享有"优先级加成"：复习排序时排在非生词之前
   - 连续 3 次评 Good 后，自动从生词本移出（"毕业"），恢复正常 SRS 调度
3. **生词本浏览**：
   - 列表视图：按添加时间/字母序/优先级排序
   - 每个词显示：拼写、释义、状态（复习中/已毕业）、在生词本中的天数
   - 支持手动移除（"我已经掌握了"）
4. **生词单独复习**：
   - 用户可主动发起"生词本专项复习"，仅复习生词本中的词
   - 专项复习的评分同样更新 SRS 间隔

#### 设计考量（融入洞察 5）

- **不孤立**：生词本不是"收藏了就再也不看"的墓地，而是自动进入 SRS 复习流的活跃队列
- **可毕业**：生词有明确的退出条件（连续 3 次 Good），给用户"我在消灭生词"的成就感
- **平均毕业周期**：7-10 天（基于数析的行业数据）

#### 验收标准

- [ ] 新词学习标记「不熟」/「不会」→ 自动加入生词本
- [ ] SRS 复习评 Again → 自动加入生词本（如尚未在生词本中）
- [ ] 拼写练习答错 → 自动加入生词本
- [ ] 用户可在生词本页面手动输入单词添加（支持英文输入 + 自动补全释义）
- [ ] 生词本列表支持按添加时间/字母序/优先级排序
- [ ] 每个生词显示：拼写、释义摘要、状态（复习中/已毕业）、在生词本天数
- [ ] 生词加入时，SRS 间隔自动设为 1 天
- [ ] 生词在 SRS 复习排序中享有优先级加成（排在非生词之前）
- [ ] 连续 3 次评 Good → 自动从生词本移出，标记为"已毕业"
- [ ] 用户可手动移除生词（标记为"已掌握"）
- [ ] 支持发起"生词本专项复习"，仅复习生词本中的词
- [ ] 专项复习的评分正常更新 SRS 间隔
- [ ] 生词本为空时显示空状态引导（"暂无生词，继续学习吧"）

#### 优先级：P0（核心收词与 SRS 融合）/ P1（手动添加、专项复习）

### 2.5 拼写 & 选择题练习

#### 功能描述

为新词学习和 SRS 复习提供**产出导向**的练习模式，强化从"再认"到"回忆"的能力：

1. **选择题练习（再认模式）**：
   - 给出中文释义，从 4 个英文单词选项中选择正确答案
   - 可作为新词学习后的即时测验，也可作为独立练习模式
   - 答错自动加入生词本
2. **拼写练习（回忆模式）**：
   - 给出中文释义 + 例句（挖空目标词），用户手动拼写英文单词
   - 拼写错误自动加入生词本
   - 支持忽略大小写和多余空格
3. **混合练习**：
   - 系统自动从今日学过/复习过的词中抽取，混合选择题和拼写题
   - 建议在当日学习完成后进行，作为巩固

#### 设计考量（融入竞析差异化机会 3）

- **再认→回忆闭环**：闪卡/选择题（再认）→ 拼写（回忆）→ 产出，逐步提升提取难度
- **不强制**：练习模式为可选项，不干扰主学习流

#### 验收标准

- [ ] 选择题：显示中文释义 + 4 个英文选项，用户点选
- [ ] 选择题答对 → 绿色高亮 + 下一题
- [ ] 选择题答错 → 红色高亮正确答案 + 该词自动加入生词本 + 下一题
- [ ] 拼写题：显示中文释义 + 挖空例句 + 输入框 + 发音按钮
- [ ] 拼写题答对 → 绿色反馈 + 下一题
- [ ] 拼写题答错 → 显示正确拼写 + 该词自动加入生词本 + 下一题
- [ ] 拼写判断忽略大小写和首尾空格
- [ ] 可发起"今日学过/复习过的词"的混合练习
- [ ] 练习完成后显示摘要：总题数、正确率、加入生词本的词数

#### 优先级：P1

### 2.6 学习统计

#### 功能描述

为用户提供有意义的进度反馈和学习洞察，核心是**"掌握感"而非"学习量"**：

1. **今日进度**（首页卡片）：
   - 今日新词学习进度（已完成/目标）
   - 今日复习进度（已完成/到期总数）
   - 今日有效学习单词数（新词 + 复习词 + 练习词）
2. **掌握度分层**（核心概念，融入洞察 4）：
   - 将所有已学单词分为四个阶段：
     - **新学**：首次学习后，尚未完成第一次 SRS 复习
     - **熟悉**：完成 1-2 次 SRS 复习，间隔 < 7 天
     - **掌握**：完成 3+ 次 SRS 复习，间隔 ≥ 7 天
     - **长期记忆**：间隔 ≥ 30 天，且最近 3 次评分无 Again
   - 首页显示各阶段词数的进度条/环形图
3. **打卡天数**：
   - 连续打卡天数（完成任意 1 个学习单元即算打卡）
   - 总打卡天数
   - 月历视图（可视化每日打卡情况）
4. **学习趋势**：
   - 近 7 天/30 天每日有效学习单词数趋势图
   - 近 7 天/30 天 SRS 评分分布（Again/Hard/Good/Easy 占比）
   - 遗忘曲线参考（基于 SRS 数据的可视化）
5. **成就里程碑**：
   - 累计掌握 100/500/1000/3000 词
   - 连续打卡 7/30/100 天
   - 生词本毕业 50/200/500 词

#### 设计考量（融入洞察 4 & 竞析差异化机会 5）

- **掌握度 > 学习量**：首屏突出"已掌握词数"，而非"已学词数"
- **有意义的洞察**：不只展示数字，还要给出建议（如"你的 Again 率偏高，建议降低每日新词量"）
- **打卡门槛低**：完成任意 1 个学习单元即算打卡（数据支撑：打卡门槛从"完成全部目标"改为"任意 1 个学习单元"→ 7 天打卡用户增加 40%）

#### 验收标准

- [ ] 首页显示今日新词进度（已完成数/目标数）
- [ ] 首页显示今日复习进度（已完成数/到期总数）
- [ ] 首页显示今日有效学习单词数
- [ ] 首页显示掌握度分层：新学/熟悉/掌握/长期记忆，各阶段词数 + 环形图
- [ ] 掌握度判定规则：新学（首次学习后未复习）、熟悉（1-2 次复习，间隔 <7 天）、掌握（3+ 次复习，间隔 ≥7 天）、长期记忆（间隔 ≥30 天 + 近 3 次无 Again）
- [ ] 连续打卡天数显示在首页显眼位置

- [ ] 成就里程碑达到时弹出庆祝提示

#### 优先级：P0（今日进度、掌握度、打卡天数）/ P1（学习趋势、成就里程碑）

    │    │    └─ 记录加入时间

    │

    │

## 4. 关键业务规则

### 4.1 SRS 评分档位说明

VocabBuddy 采用四档评分体系，基于数析提供的参数和间隔重复研究（Kornell 2009, Cepeda 2006）：

| 评分 | 含义 | 间隔变化规则 | 稳态期占比 | 用户场景 |
|------|------|------------|-----------|---------|
| **Again** | 完全忘记，无法回忆 | **重置为 1 天**（次日再现） | 10%-15% | 翻转前完全想不起来 |
| **Hard** | 勉强记得，回忆吃力 | **当前间隔 × 1.2**（向上取整，最少 +1 天） | 15%-20% | 翻转前犹豫很久才想起 |
| **Good** | 正常记得，有短暂犹豫 | **当前间隔 × 2.5**（向上取整，标准增长） | 55%-65% | 翻转后确认"对，就是这个" |
| **Easy** | 秒答，毫无犹豫 | **当前间隔 × 3.0~4.0**（向上取整，具体系数后续定） | 10%-15% | 不用翻卡就知道答案 |

**补充规则**：
- **新词首次评分**（新词学习阶段）：标记「认识」→ 间隔 4 天；标记「不熟」/「不会」→ 间隔 1 天
- **最小间隔**：任何情况下间隔不少于 1 天
- **最大间隔**：单次间隔不超过 180 天（超过 180 天的词视为"长期记忆"，不再频繁调度）
- **间隔取整**：所有计算结果向上取整为整数天
- **连续 Again 处理**：同一词连续 2 次评 Again，系统在第 3 次出现时增加提示（"这个词你可能需要更多关注"）

**预警阈值**（来自数析）：
- Again 率 > 20% → 黄色预警，提示用户降低每日新词量
- Again 率 > 30% → 红色预警，系统自动建议暂停新词学习

> ⚠️ **具体算法后续定**：以上为 MVP 阶段的简化规则。未来可引入 FSRS 等更精细的算法模型，根据用户历史评分数据个性化调整间隔倍数。MVP 先用固定参数跑通流程，后续版本加入自适应参数。

### 4.2 新词 vs 复习词判定条件

| 维度 | 新词 | 复习词 |
|------|------|--------|
| **定义** | 当前词库中尚未学习过的词 | 已学过且 SRS 队列中 next_review_date ≤ today 的词 |
| **判定条件** | `word.status == 'unlearned'` | `word.status != 'unlearned' AND word.next_review_date <= today` |
| **来源** | 当前选中词库的未学部分 | SRS 调度队列 |
| **每日拉取量** | 用户设定的每日新词量（默认 15） | 当日所有到期词（受防雪崩和上限约束） |
| **交互模式** | 闪卡翻转 + 三档标记（认识/不熟/不会） | 闪卡翻转 + 四档评分（Again/Hard/Good/Easy） |
| **首次间隔** | 认识→4天，不熟/不会→1天 | 由评分决定（见 4.1） |

**特殊状态词**：
- **已毕业的生词**：仍属于复习词，按正常 SRS 调度
- **手动添加的生词**：首次进入时视为新词（首次复习间隔 = 1 天），之后按复习词处理
- **词库切换后**：原词库的学习进度保留，新词库从第一个未学词开始

### 4.3 生词判定条件

一个词在以下任一条件下被判定为"生词"并加入生词本：

| 触发条件 | 来源场景 | 生词本 priority |
|---------|---------|----------------|
| 新词学习中用户标记「不熟」 | 新词学习流程 | normal |
| 新词学习中用户标记「不会」 | 新词学习流程 | high |
| SRS 复习中用户评 Again | 复习流程 | normal |
| 拼写练习答错 | 拼写练习 | normal |
| 选择题答错 | 选择题练习 | normal |
| 用户手动添加 | 生词本页面 | normal |

**生词退出条件**（满足任一）：
1. **自动毕业**：在 SRS 复习中连续 3 次评 Good → 自动移出生词本
2. **手动移除**：用户在生词本中点击"已掌握"→ 移出生词本

**生词率合理区间**（来自数析，用于监控和预警）：
- 新词首次学习后加入生词本比例：25%-40%
- SRS 复习中 Again 占比：10%-15%
- 生词本中词平均毕业周期：7-10 天

### 4.4 每日学习量管理规则

基于数析的数据分析报告，制定以下每日学习量管理规则：

#### 4.4.1 新词量规则

| 规则 | 数值 | 数据依据 |
|------|------|---------|
| 默认每日新词量 | 15 个 | Nakata 2015：每日 15 词→一周后保持率 73%；每日 40 词→42% |
| 用户可调范围 | 10-25 个 | 兼顾效率和负荷 |
| 调节步进 | 5 个 | — |
| 当日复习 > 80 时 | 系统建议临时降至 10 个 | 防止总负荷过高 |
| 稳态期新词:复习比 | 1:5~1:7 | 第 5 周起达到稳态 |

#### 4.4.2 复习量管理

| 规则 | 数值 | 说明 |
|------|------|------|
| 默认每日复习上限 | 100 个 | 用户可调 50-200 |
| 防雪崩触发阈值 | 80 个 | 到期词 > 80 时启动分摊 |
| 分摊策略 | 超出部分分摊到未来 2-3 天 | 按遗忘概率排序，优先复习高风险词 |
| 分摊时新词处理 | 建议暂停新词 | 用户可选择忽略建议 |
| 复习优先模式 | 复习 > 50 时暂停新词 | 用户可开启/关闭 |

#### 4.4.3 总负荷控制

- **总学习量** = 当日新词 + 当日复习词 + 当日练习题
- **建议上限**：每日总学习量不超过 150 个单词交互（约 20-30 分钟）
- **超限提醒**：当总学习量接近上限时，提示用户"今日学习量较大，注意劳逸结合"

### 4.5 打卡与断链恢复规则

基于瑞思的洞察 3 和数析的打卡数据，设计柔性断链恢复机制：

#### 4.5.1 打卡规则

| 规则 | 说明 | 数据依据 |
|------|------|---------|
| 打卡判定 | 完成任意 1 个学习单元（新词学习/复习/练习）即算当日打卡 | 打卡门槛从"完成全部目标"改为"任意 1 个单元"→ 7 天打卡用户 +40% |
| 连续打卡 | 连续天数自动计算，显示在首页 | 7 天打卡用户次日留存 = 未打卡 × 2.4 倍 |
| 月历视图 | 可视化每月打卡情况 | 增强坚持动机 |

#### 4.5.2 断链恢复机制（柔性断链保护）

这是 VocabBuddy 的核心差异化之一，解决"难以坚持"痛点：

| 场景 | 机制 | 说明 |
|------|------|------|
| **断链 1-3 天** | 积压分摊 | 期间累积的到期复习词不会一次性涌出，而是自动分摊到未来 3-5 天，首日负荷降低 50% |
| **断链 4-7 天** | 智能筛选 + 分摊 | 按"最可能遗忘"排序，优先复习高风险词；积压分摊到未来 5-7 天；首日仅复习最紧急的 30 个 |
| **断链 7+ 天** | 温和重启 | 系统提示"欢迎回来！已为您重新安排复习计划"；积压分摊到未来 7-10 天；首日仅复习最紧急的 20 个 + 建议暂停新词 |
| **每月补卡机会** | 2 次 | 用户每月有 2 次"冻结/补卡"机会，可保持连续打卡天数不中断 |

**断链回归首日降低负荷的具体规则**：
- 回归首日复习量 = min(当日到期词, 正常每日上限 × 50%)
- 回归首日新词量默认暂停（用户可手动恢复）
- 回归首日显示"欢迎回来"引导，不展示积压数字（避免焦虑）

**补卡机制**：
- 每月 2 次补卡机会，用于保持连续打卡天数
- 补卡不补学习内容（不会要求用户补学之前的内容）
- 补卡后连续天数保持不变

> ⚠️ **假设**：断链恢复的具体分摊参数（分摊天数、首日负荷比例）为 MVP 阶段的初始设定，上线后需根据实际用户行为数据调优。

## 7. 用户研究洞察（来自瑞思）

以下 5 条关键洞察指导了 VocabBuddy 的核心设计决策：

### 洞察 1：「主动提取」比「被动重复」更有效

> SRS 复习必须有"先考后看"环节：先呈现刺激（单词）→ 用户回忆释义 → 翻转揭晓答案 → 用户自评。

**设计落地**：
- SRS 复习闪卡正面仅显示单词，不显示释义，强制用户先回忆
- 翻转前有短暂等待，鼓励提取过程
- 不提供"直接看答案"的捷径

### 洞察 2：碎片化场景下「单次学习可中断」是刚需

> 随时退出、进度自动保存、下次续接；单词学习周期 ≤30 秒，2 分钟也能完成有意义交互。

**设计落地**：
- 每个单词学习后进度即时保存到本地
- 随时可退出，下次从断点续接
- 首页显示今日进度，进入即学、退出即走

### 洞察 3：断链重启体验决定长期留存

> 积压自动分摊到未来几天（非一次性涌出）；连续天数提供容错/补签；断链回归首日降低负荷。

**设计落地**：
- 柔性断链恢复机制（4.5.2）：积压分摊 + 首日降负荷 + 欢迎回归引导
- 每月 2 次补卡机会保持连续天数
- 回归首日不展示积压数字，避免焦虑

### 洞察 4：用户需要"掌握感"而非"学习量"

> 单词分"新学→熟悉→掌握→长期记忆"阶段，核心指标是"已掌握词数"而非"已学词数"。

**设计落地**：
- 掌握度分层体系（2.6）：四阶段可视化
- 首页突出"已掌握词数"，北极星指标为"日均有效学习单词数"
- 统计页提供有意义的洞察，而非堆砌数字

### 洞察 5：生词本必须与 SRS 融合

> 标记生词自动提高复习频率/缩短间隔，融入每日复习流；连续 3 次 Good 后自动移出。

**设计落地**：
- 生词本不是独立列表，而是 SRS 队列的活跃子队列
- 生词享有优先级加成（复习排序靠前）
- 自动毕业机制（连续 3 次 Good 移出）
- 平均毕业周期 7-10 天，给用户"消灭生词"的成就感

## 11. Non-goals（明确不做什么）

> 非目标和目标同样重要。以下内容在 MVP 阶段明确不做，任何范围变更需经评审。

| Non-goal | 理由 |
|---------|------|
| **不做社交功能** | 与"专注学习、零社交噪音"的产品理念冲突；社交功能开发复杂且分散焦点 |
| **不做 AI 例句生成** | 预置例句已满足 MVP 需求；AI 生成质量不稳定且增加 API 成本 |
| **不做多端云同步** | 账号体系 + 数据冲突解决复杂度高；MVP 聚焦单端体验验证 |
| **不做会员/付费体系** | 先验证产品价值，不引入商业化干扰 |
| **不做 UGC 词库市场** | 词库质量控制成本高；MVP 预置词库覆盖主要场景 |
| **不做广告** | 零广告是核心差异化（对比百词斩广告多被差评） |
| **不做图片联想记忆** | 图片制作成本高且覆盖不全；MVP 用例句语境替代 |
| **不做游戏化通关** | 与"清爽专注"理念冲突；游戏化容易导致"为了通关而学习"而非真实掌握 |
| **不做生态联动**（阅读/听力/口语） | 扇贝模式功能碎片化；MVP 聚焦单词学习本身 |
| **不做 SRS 算法完全透明/可调参数** | Anki 的参数开放性导致新手劝退；MVP 用四档简化评分，进阶调参留到 v2 |
